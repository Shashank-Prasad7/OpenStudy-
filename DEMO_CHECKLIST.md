# OpenStudy five-minute demo checklist

## Before recording

1. Start the complete stack and confirm `/health` reports the database and Redis as available.
2. Create two accounts in separate browser profiles or an incognito window.
3. Pre-create one goal and one completed session so the dashboard is populated.
4. Keep both browser windows visible side by side.
5. Confirm the backend, frontend, repository, and architecture slide are already open.

## Demo sequence

1. Register or log in and show the live dashboard.
2. Create a room with subject tags and copy its direct link.
3. Open the link in the second signed-in browser and join.
4. Show the live member list update in both windows.
5. Start the Pomodoro and show synchronized countdowns.
6. Pause/resume or reset, then demonstrate the completion note prompt.
7. Add and complete a goal; show the streak/statistics refresh.
8. Save matching preferences and show ranked accountability partners.
9. Generate a study plan for an upcoming exam.
10. End on the repository CI, documentation, MIT license, and architecture diagram.

## Final checks

- No secrets appear on screen.
- Browser zoom is readable at 1080p.
- Notifications and loading delays have been rehearsed.
- Captions are enabled in the final recording.
